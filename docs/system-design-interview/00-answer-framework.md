# Answer Framework

## Base structure

Use this order:

1. context
2. choice
3. tradeoff
4. risk
5. when I would change the decision

## Short formula

`I would choose X because the problem needs Y. The main gain is Z. The cost is W. If the context changed to K, I would reconsider A.`

## What makes the answer sound senior

- it does not follow trends blindly
- it explains operational cost
- it talks about scale and complexity together
- it makes clear that every choice closes some doors

## What to avoid

- giving only the textbook definition
- saying better or worse without context
- using microservices, event-driven systems, or cache as automatic answers
- dropping buzzwords without explaining why

## Useful lines

- `It depends more on the access pattern than on the technology itself.`
- `I would try to delay distributed complexity until there is real pressure for it.`
- `The bottleneck needs to be measured before it gets optimized.`
- `I separate product problems from infrastructure problems.`
