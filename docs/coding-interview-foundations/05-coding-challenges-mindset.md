# Coding Challenges Mindset

## Important truth

These problems are not testing whether you are a genius.
They are mostly testing:

- pattern recognition
- calmness under pressure
- ability to avoid repeated work

## Better mindset

Instead of:

- `I do not know this exact problem`

Think:

- `What type of repeated work is this problem trying to trick me into doing?`

## Good practice rules

- start easy
- solve fewer problems but understand them deeply
- write the brute-force version first if needed
- explain the improvement in plain English

## What not to do

- jump only into medium and hard problems
- compare yourself to competitive programmers
- memorize code without understanding the pattern

## Real target

You do not need to become a competitive programming person.
You need to become someone who can recognize common patterns fast enough in interviews.
