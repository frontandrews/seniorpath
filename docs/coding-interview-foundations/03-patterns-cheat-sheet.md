# Patterns Cheat Sheet

## Array / string + repeated lookup

Think:

- hashmap
- set

## Sorted array

Think:

- two pointers
- binary search

## Longest / shortest valid contiguous piece

Think:

- sliding window

## Nested or reverse closing behavior

Think:

- stack

## First in, first out behavior

Think:

- queue

## Need to cut the search space in half

Think:

- binary search

## Counting or comparing frequencies

Think:

- frequency map

## Real rule

The pattern is not chosen because the interviewer likes it.
The pattern is chosen because it removes repeated work.
